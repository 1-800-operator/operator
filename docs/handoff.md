# Session 184 handoff (2026-05-04) — cloud-browser architecture validated; sidecar replaces plugin at L2

This was a strategy + spike session. **Two doc commits on local main (`accc019` and the upcoming Hard Won Knowledge commit), no production code changes.** The session-183 plugin path was killed and replaced with a four-tier trust ladder: **L0 (free)** = paste link + email, cloud bot joins, 30 min/account, generic Haiku model, phantom-action footers tease L2; **L1 (paid)** = $19/mo Stripe subscription, more hours, Sonnet 4.6, cloud-only; **L2 (sidecar)** = signed/notarized macOS app downloaded from `1-800-operator.com/download`, bridges cloud bot to user's local Claude Code (skills, MCPs, Linear, GitHub, codebase) — *replaces the dead Claude Code plugin from session 183*; **L3 (Brainchild)** = v2 build-your-own-bot. Eight spike runs against real Meet rooms on existing DigitalOcean droplet `operator-dev` (`64.23.182.26`, $12/mo always-on, alive since March) validated the cloud-browser architecture but surfaced **the load-bearing finding: anonymous-guest cloud bots cannot maintain WebRTC from a datacenter IP — Google drops them in ~10s, chat input renders in the bot's local UI but never broadcasts. Authenticated bots (Playwright `storage_state` from a saved Google session) sustain WebRTC indefinitely and deliver chat (Run 8 confirmed: user received `[droplet probe] hello from cloud at 2026-05-04T05:41:36+00:00` chat message). Therefore: every cloud bot in v1 must run as a real Google account.** Privacy flow: at signup, user creates a fresh Google account *just for the bot*, signs in via a hosted-browser flow on our server (noVNC-streamed Chromium), we capture and store only the encrypted session cookies — never the password, never OAuth scopes for personal Google data. `docs/product-strategy.md` was rewritten end-to-end with the new ladder, A1–A10 phases, validated cloud-browser config, the auth/privacy section, and a three-candidate spike plan for production fleet (Google Cloud Run / AWS Fargate / Fly.io — Netlify ruled out as structurally wrong; DO ruled out for cost shape with a confirming $1.29 idle-billing observation). New memory `project_cloud_browser_auth_requirement.md` captures the spike methodology + auth flow; `project_closed_source_pivot` and `project_v1_track_a_only` updated; `MEMORY.md` index reflects all changes.
