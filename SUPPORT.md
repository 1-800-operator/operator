# Getting help

## Found a bug?

[**Open an issue**](https://github.com/1-800-operator/operator/issues/new/choose) using the **Bug report** template. Fill in the OS, install method, command you ran, and the relevant slice of `/tmp/operator.log` — the more specific, the faster a fix. Redact secrets and meeting URLs before pasting logs.

## Have a question, idea, or want to show off what you built?

[**Use GitHub Discussions**](https://github.com/1-800-operator/operator/discussions). Categories:

- **Q&A** — anything starting with "how do I…" or "is it possible to…".
- **Ideas** — feature thoughts that aren't concrete enough yet for an issue.
- **Show & tell** — agents you've built, MCPs you've wired up, fun meeting outcomes.
- **General** — everything else.

The issue tracker is reserved for bugs and concrete feature requests so it stays useful for triage.

## Found a security vulnerability?

**Do not file a public issue.** Follow [`SECURITY.md`](./SECURITY.md) — email or open a private GitHub Security Advisory. Acknowledgement within 72 hours.
